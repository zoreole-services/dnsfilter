# DNS Filter

**DNS Filter**  is a  [Python](https://www.python.org/)  package that retrieves a list of domains from a file stored in an  [S3](https://aws.amazon.com/s3/)  bucket using the  [Boto3](https://boto3.amazonaws.com/v1/documentation/api/latest/index.html)  library, filters them, and converts them into a rpz file that will be taken in account the DNS servers.

**DNS Filter**  operates within a  [Docker](https://www.docker.com/)  container, making it easy to deploy and manage.

---

## Operating Modes

DNS Filter supports three modes, depending on the DNS infrastructure:

| Mode | Description |
|------|--------------|
| **BINDLOCAL** | The solution directly generates an RPZ (Response Policy Zone) file, which is locally read by the Bind9 process on the **host**. |
| **BINDTRANSFER** | The solution generates the RPZ file and serves it using a Bind9 instance **running inside the container**, acting as the **master server** for the RPZ zone that can be queried by Bind9 slave servers. |
| **BLUECATAPI** | The container connects to a **BlueCat BAM** server through its REST API (v2) and creates or updates RPZ entries directly within BlueCat. |

## Prerequisites

Before installation, ensure the following components are available:

- **Docker** installed on your system  
  - [Install Docker on Debian](https://docs.docker.com/engine/install/debian/)  
  - [Install Docker on Rocky Linux](https://docs.rockylinux.org/gemstones/containers/docker/)
  - Docker Compose, for easier orchestration of services

## Installation

1. **Clone the repository**

   ```bash
   git clone https://github.com/zoreole-services/dnsfilter.git
   cd dnsfilter
   ```

2. **Prepare your Docker Compose file**

   Copy the example file and adapt it to your setup:

   ```bash
   cp docker-compose.yaml.example docker-compose.yaml
   ```

   Then edit `docker-compose.yaml` to configure environment variables, ports, and volumes.

## Configuration

### Common Environment Variables

| Variable | Default | Required | Description |
|-----------|----------|-----------|-------------|
| `AWS_ACCESS_KEY_ID` | - | Yes | AWS access key for S3 bucket |
| `AWS_SECRET_ACCESS_KEY` | - | Yes | AWS secret access key |
| `AWS_REGION` | `eu-west-1` | No | AWS region |
| `S3_BUCKET_NAME` | - | Yes | Name of S3 bucket |
| `S3_OBJECT_KEY_PATH` | - | Yes | Path to object in S3 |
| `S3_OBJECT_FILE_NAME` | - | Yes | File name containing domains |
| `EXECUTION_INTERVAL` | `30` | No | Polling interval (seconds) |
| `SOLUTION_IDENTIFIER` | - | Yes | Solution names |
| `DNS_TTL` | `60` | No | DNS entries TTL |
| `DOMAIN_WHITELIST` | - | No | Domain entries to whitelist |



---

### Solution 1: BINDLOCAL

The RPZ file generated by the script is read directly by a **Bind9 instance running on the host**.



**Crontab Example:**
```bash
*/30 * * * * bash /path/of/dnsfilter/folder/scripts/reload_bind.sh
```

N.B : Please replace "/path/of/dnsfilter/folder/" by the current folder path of the DNSFilter project.

---

### Solution 2: BINDTRANSFER

In this mode, Bind9 runs **inside the container**, acting as **master** for the RPZ zone.  
Other DNS servers can perform zone transfers from this master.


| Variable | Default | Required | Description |
|-----------|----------|-----------|-------------|
| `BIND_SLAVE_IPADDR` | - | Yes* | DNS Slave IP address |

* Those variables are only mandatory if BINDTRANSFER solution is selected.


**Bind Configuration:**

In `/etc/bind/named.conf.local`:
```bash
zone "rpz" {
    type master;
    file "/etc/bind/zones/rpz.db";
    allow-query { none; };
};
```

In `/etc/bind/named.conf.options`:
```bash
recursion yes;
allow-query { any; };
response-policy { zone "rpz"; };
```

---

### Solution 3: BLUECATAPI

This mode connects to a **BlueCat BAM** server using its **REST API v2**, and update its response policies zone.

| Variable | Default | Required | Description |
|-----------|----------|-----------|-------------|
| `BLUECAT_USER` | - | Yes* | BlueCat BAM username |
| `BLUECAT_PWD` | - | Yes* | BlueCat BAM password |
| `BLUECAT_IPADDR` | - | Yes* | BlueCat BAM IP address |
| `BLUECAT_TARGET_BDDS` | - | Yes* | Target BDDS server name |
| `BLUECAT_RPZONE_NAME` | - | No | Response Policy Zone name |


* Those variables are only mandatory if BLUECATAPI solution is selected.

**BlueCat Configuration:**

Ensure an **Response Policy Zone** exists in *DNS > DNS View > RP Zones* and that the policy created by the script (default: `dnsfilter`) is attached to it.

## Usage

**Start the container:**
```bash
docker compose up -d
```

**Stop the container:**
```bash
docker compose down
```

**Rebuild the image:**
```bash
docker compose build dnsfilter
docker compose down && docker compose up -d
```
